# VK-уведомления о колёсах

VK используется только как дополнительный канал доставки уведомлений о новых колёсах BetBoom. Управление ботом, аналитика, настройки и административные действия остаются в Telegram.

## Схема

1. Основной монитор формирует обычное уведомление о новом колесе и доставляет его в Telegram через действующий `notification_router`.
2. `vk_wheel_notifications.py` распознаёт только пользовательский тип `wheels`; финальные напоминания и уведомления о времени прокрутки во VK не маршрутизируются.
3. После Telegram-доставки модуль запускает отдельный workflow `vk-wheel-notification.yml` через GitHub Actions API.
4. Workflow отправляет текст и ссылку методом VK API `messages.send`.
5. Ошибка VK не меняет результат Telegram-доставки и не останавливает монитор.

## Секреты GitHub Actions

Для фактической отправки необходимо создать два repository secrets:

- `VK_GROUP_TOKEN` — токен сообщества VK с правом работы с сообщениями;
- `VK_WHEEL_PEER_IDS` — один или несколько `peer_id`, разделённых запятыми, пробелами или `;`.

Пока секреты не заданы, VK-workflow завершается успешно без отправки. Telegram продолжает работать без изменений.

## Дедупликация

Запуск VK-доставки использует существующий журнал дедупликации уведомлений с отдельным адресатом `vk:wheel-notifications`. На стороне VK также передаётся детерминированный `random_id`, вычисленный из идентичности события и `peer_id`.

## Откат

Удалить установку `vk_wheel_notifications` из `bbvg_monitor_main.py` и workflow `.github/workflows/vk-wheel-notification.yml`. Telegram-маршрутизация при этом не требует миграции состояния.
