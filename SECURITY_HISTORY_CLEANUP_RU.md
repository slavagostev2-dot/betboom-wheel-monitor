# Контролируемая очистка Git-истории

Эта инструкция не выполняется автоматически. Переписывание истории удаляет
старые commit hash и требует повторного клонирования репозитория всеми
участниками.

## Условия перед началом

1. Панель и монитор работают с `bbvg-bot-state-v2`.
2. В `admin_panel_status.json` указано `state_key_mode: dedicated`.
3. Сделана резервная копия `bot_private_state.enc.json`.
4. Резервная копия успешно расшифровывается через `BOT_STATE_KEY`.
5. Проверены пользователи, роли, уведомления и заявки.
6. Владелец отдельно подтвердил разрушительную очистку истории.

## Инвентаризация

```bash
python security_audit.py --current
python security_audit.py --history --history-output security_history_report.json
```

Исторический отчёт содержит только имена файлов и сокращённые commit hash.

## Резервная точка

Перед фильтрацией создать локальный bare-clone и защищённый tag/архив. Архив с
персональными данными не публиковать и хранить только в контролируемом месте.

## Фильтрация

Рекомендуемый инструмент — `git filter-repo`. Нужно удалить содержимое старых
версий `bot_access.json` и `source_requests.json` либо заменить их безопасными
пустыми шаблонами во всей истории. Конкретный callback-фильтр подготавливается
после просмотра отчёта и тестируется в отдельном клоне.

После фильтрации обязательно выполнить повторно:

```bash
python security_audit.py --current
python security_audit.py --history --fail-on-history
```

## Публикация переписанной истории

Только после проверки:

1. остановить долгоживущие GitHub Actions;
2. защитить актуальное зашифрованное состояние;
3. выполнить force-push всех затронутых веток и tags;
4. удалить устаревшие ветки и pull request refs, где это возможно;
5. перезапустить production workflows;
6. проверить панель, монитор и уведомления;
7. потребовать от локальных участников удалить старые клоны и клонировать заново.

## Ограничения

Переписывание истории не гарантирует удаление данных из уже созданных forks,
локальных клонов, сторонних caches и ранее скачанных архивов. Если в истории
когда-либо находился секрет, его нужно отозвать независимо от очистки Git.
