# Перенос личного состояния в Cloudflare D1

Этот документ относится к Worker из `state_api/`, панели `admin_panel_runtime_v23.py` и Mini App.

## Автоматическая последовательность после объединения PR

1. Workflow `state-api.yml` создаёт или находит D1-базу `bbvg-private-state` в юрисдикции `eu`.
2. Создаются таблицы пользователей, ролей, настроек уведомлений, участий, скрытых колёс, пользовательских настроек и заявок источников.
3. Worker разворачивается с закрытым `BOT_TOKEN` и записывает свой публичный HTTPS URL в `private_state_deployment.json`.
4. Запускается отдельный workflow `migrate-private-state.yml`. Он находит последнюю непустую версию `bot_access.json` и `source_requests.json` в Git-истории и переносит записи в D1. В лог выводятся только количества записей.
5. После миграции перезапускаются Mini App, Telegram-панель и основной монитор.
6. Панель `v23` не запускается, пока приватный API не получил статус `deployed`.

## Проверка перед очисткой истории

Историю нельзя переписывать до подтверждения миграции. Сначала должны выполняться все условия:

- workflow развёртывания Worker завершён успешно;
- workflow миграции завершён успешно;
- существующие владелец и администраторы видны в панели;
- новый пользователь после открытия Mini App появляется в панели без сообщения боту;
- отметка «Я участвую» одинакова в Mini App и боте;
- скрытое колесо остаётся скрытым после повторного открытия;
- заявка источника появляется у администратора, а Mini App показывает подтверждение только после ответа API.

## Очистка Git-истории

Очистка истории — отдельная разрушительная операция: меняются SHA затронутых коммитов, открытые ветки и локальные клоны потребуется синхронизировать заново. Выполнять её следует только после проверки D1 и резервного экспорта базы.

На чистом локальном клоне с установленным `git-filter-repo`:

```bash
git clone --mirror https://github.com/slavagostev2-dot/betboom-wheel-monitor.git
cd betboom-wheel-monitor.git

git filter-repo \
  --path bot_access.json \
  --path source_requests.json \
  --invert-paths \
  --force

git push --force --mirror
```

После переписывания истории нужно заново добавить в актуальный `main` обезличенные шаблоны `bot_access.json` и `source_requests.json`, проверить обязательные workflow и пересоздать старые локальные клоны.

Удаление данных из текущей ветки и переписывание основного репозитория не удаляет уже созданные форки, архивы, клоны и внешние кэши.
